# types.md — система типов языка **Astra**

## 1. Назначение документа

Документ определяет систему типов языка **Astra**, правила совместимости,
вывода типов, приведения и проверки типовой корректности программ.

Система типов языка соответствует обязательным требованиям ТЗ:

- статическая строгая типизация;
- номинальная типизация;
- отсутствие неявных приведений;
- value semantics;
- поддержка встроенных типов, массивов фиксированного размера, структур и type alias.

---

## 2. Категории типов

В языке существуют следующие категории типов:

1. **целочисленные знаковые**
2. **целочисленные беззнаковые**
3. **вещественные**
4. **булев тип**
5. **строки**
6. **unit**
7. **массивы фиксированного размера**
8. **структуры**
9. **синонимы типов (type alias)**

---

## 3. Встроенные типы

### 3.1. Целочисленные знаковые типы

| Имя типа | Размер | Диапазон значений |
|----------|--------|-------------------|
| `int8`   | 8 бит  | -128 .. 127 |
| `int16`  | 16 бит | -32768 .. 32767 |
| `int32`  | 32 бит | -2^31 .. 2^31-1 |
| `int64`  | 64 бит | -2^63 .. 2^63-1 |

### 3.2. Целочисленные беззнаковые типы

| Имя типа | Размер | Диапазон значений |
|----------|--------|-------------------|
| `uint8`  | 8 бит  | 0 .. 255 |
| `uint16` | 16 бит | 0 .. 65535 |
| `uint32` | 32 бит | 0 .. 2^32-1 |
| `uint64` | 64 бит | 0 .. 2^64-1 |

### 3.3. Вещественные типы

| Имя типа  | Размер | Формат |
|-----------|--------|--------|
| `float32` | 32 бит | IEEE 754 single |
| `float64` | 64 бит | IEEE 754 double |

### 3.4. Булев тип

| Имя типа | Значения |
|----------|----------|
| `bool`   | `true`, `false` |

### 3.5. Строки

| Имя типа | Описание |
|----------|----------|
| `string` | последовательность байтов UTF-8 |

Строка — неизменяемое значение. Любая операция над строкой создаёт новое значение.

### 3.6. Пустой тип результата

| Имя типа | Описание |
|----------|----------|
| `unit`   | отсутствие полезного значения |

`unit` используется как тип результата функций, не возвращающих осмысленное значение.

---

## 4. Составные типы

### 4.1. Массив фиксированного размера

Тип массива имеет вид:

```txt
[T; N]
```

где:

- `T` — тип элемента;
- `N` — положительная целочисленная константа, являющаяся частью типа.

Примеры:

```txt
[int32; 4]
[bool; 10]
[string; 2]
```

#### Свойства массива

- размер известен на этапе компиляции;
- массив передаётся и присваивается по значению;
- два массива совместимы только если совпадают и тип элемента, и размер.

Следовательно:

- `[int32; 3]` и `[int32; 4]` — разные типы;
- `[int32; 3]` и `[int64; 3]` — разные типы.

### 4.2. Структуры

Структура — именованный пользовательский составной тип.

Пример:

```txt
struct Point {
    x: int32;
    y: int32;
}
```

Тип структуры определяется её именем и пространством имён.

Поля структуры образуют фиксированную схему.

### 4.3. Синонимы типов

`type Alias = ExistingType;`

Синоним типа не создаёт новый тип.

Пример:

```txt
type Count = int32;
```

`Count` и `int32` являются одним и тем же типом в смысле совместимости.

---

## 5. Номинальная типизация

Язык использует **номинальную типизацию**.

Это означает, что два типа совместимы только если их имена и структура тождественно совпадают
с учётом правил alias.

### 5.1. Следствия номинальной типизации

1. Разные структуры с одинаковым набором полей — разные типы.
2. Структурная совместимость не применяется.
3. Полное имя типа учитывает пространство имён.

Пример:

```txt
namespace A {
    struct Point {
        x: int32;
        y: int32;
    }
}

namespace B {
    struct Point {
        x: int32;
        y: int32;
    }
}
```

`A::Point` и `B::Point` несовместимы.

### 5.2. Исключение для type alias

Type alias не создаёт новый тип.

Следовательно:

```txt
type Meters = int32;
```

`Meters` совместим с `int32` без дополнительных преобразований.

---

## 6. Правила совместимости типов

### 6.1. Тождественная совместимость

Типы совместимы, если после раскрытия цепочки alias они одинаковы.

### 6.2. Числовые типы

Числовые типы **не совместимы неявно**.

Например:

- `int32` нельзя присвоить в `int64` без `as`;
- `uint32` нельзя передать туда, где ожидается `int32`;
- `int32` нельзя использовать там, где ожидается `float64`, без явного приведения.

### 6.3. Структуры

Совместимы только значения одной и той же структуры.

### 6.4. Массивы

Совместимы только массивы одного и того же типа элемента и размера.

### 6.5. Строки и bool

`string` и `bool` совместимы только сами с собой.

### 6.6. Unit

`unit` совместим только с `unit`.

Его нельзя использовать как обычное значение в арифметике, присваивании или сравнении,
если спецификация конкретной операции это явно не разрешает.

---

## 7. Типизация переменных

### 7.1. Объявление `var`

`var` требует явного указания типа:

```txt
var x: int32 = 10;
```

Тип инициализатора обязан точно совпадать с объявленным типом переменной.

### 7.2. Объявление `let`

У `let` возможны два режима.

#### Явный тип

```txt
let x: int32 = 10;
```

Инициализатор обязан иметь тип `int32`.

#### Вывод типа

```txt
let x = 10;
```

Тип переменной выводится как тип инициализатора.

Это дополнительная возможность языка сверх обязательной базы.

---

## 8. Правила вывода типов

### 8.1. Общий принцип

Вывод типа в языке **Astra** локальный и одношаговый.

Компилятор не решает сложные системы ограничений.
Он просто берёт уже известный тип инициализатора и присваивает его переменной `let`.

### 8.2. Вывод по литералам

```txt
let a = 10;       // int32
let b = 3.14;     // float64
let c = true;     // bool
let d = "text";  // string
```

### 8.3. Вывод по выражениям

```txt
let x = 1 + 2;        // int32
let y = foo();        // тип результата foo
let z = Point { x: 1, y: 2 }; // Point
```

### 8.4. Вывод массива

```txt
let xs = [1, 2, 3];
```

Тип: `[int32; 3]`.

Все элементы должны иметь один и тот же тип.

### 8.5. Ограничения вывода

Вывод запрещён, если:

- тип инициализатора неоднозначен;
- литерал массива пуст и нет ожидаемого типа;
- выражение содержит ошибку типов.

---

## 9. Типы литералов

### 9.1. Целые литералы

Целый литерал без контекста имеет тип `int32`.

Пример:

```txt
let x = 42; // int32
```

Если литерал используется в контексте, требующем другой тип,
неявная адаптация **не выполняется**.

Следовательно, это ошибка:

```txt
var x: int64 = 42;
```

а это корректно:

```txt
var x: int64 = 42 as int64;
```

### 9.2. Вещественные литералы

Вещественный литерал без контекста имеет тип `float64`.

### 9.3. Булевы литералы

`true` и `false` имеют тип `bool`.

### 9.4. Строковые литералы

Строковый литерал имеет тип `string`.

---

## 10. Типизация выражений

### 10.1. Унарные операции

| Операция | Допустимые типы | Результат |
|----------|------------------|-----------|
| `-x`     | любой числовой тип | тот же тип |
| `!x`     | `bool` | `bool` |

### 10.2. Бинарные арифметические операции

| Операция | Допустимые типы | Условие | Результат |
|----------|------------------|---------|-----------|
| `+` | числовые, `string` | типы операндов совпадают | тот же тип |
| `-` | числовые | типы операндов совпадают | тот же тип |
| `*` | числовые | типы операндов совпадают | тот же тип |
| `/` | числовые | типы операндов совпадают | тот же тип |
| `%` | целочисленные | типы операндов совпадают | тот же тип |

Для строк `+` означает конкатенацию и имеет тип `string`.

### 10.3. Логические операции

| Операция | Операнды | Результат |
|----------|----------|-----------|
| `&&` | `bool`, `bool` | `bool` |
| `||` | `bool`, `bool` | `bool` |

### 10.4. Операции сравнения

#### Равенство / неравенство

`==` и `!=` требуют одинаковые типы операндов.

Допустимые типы:

- любые числовые типы при точном совпадении типов;
- `bool`;
- `string`;
- массивы одного и того же типа;
- структуры одного и того же типа.

Результат — `bool`.

#### Порядковые сравнения

`<`, `<=`, `>`, `>=` допускаются только для числовых типов при точном совпадении типов.

Результат — `bool`.

### 10.5. Индексирование

Если `arr: [T; N]` и `i` — целочисленный тип,
то `arr[i]` имеет тип `T`.

### 10.6. Доступ к полю

Если `s: S`, а `S` — структура с полем `f: T`,
то `s.f` имеет тип `T`.

### 10.7. Вызов функции

Если функция имеет тип

```txt
fn (T1, T2, ..., Tn) -> R
```

то выражение вызова имеет тип `R`,
если все аргументы имеют точно типы `T1..Tn`.

### 10.8. Литерал структуры

Если `S` — объявленная структура,
то литерал `S { ... }` имеет тип `S`.

### 10.9. Оператор `as`

Если приведение разрешено правилами раздела 11,
то `expr as T` имеет тип `T`.

---

## 11. Явные приведения типов

### 11.1. Общий принцип

Неявных приведений в языке нет.
Любое преобразование между различными типами должно быть записано явно через `as`.

### 11.2. Разрешённые приведения

#### Между целыми типами

Разрешены все приведения между:

- `int8`, `int16`, `int32`, `int64`
- `uint8`, `uint16`, `uint32`, `uint64`

#### Между вещественными типами

Разрешены приведения:

- `float32 <-> float64`

#### Между целыми и вещественными

Разрешены приведения:

- любой целочисленный -> любой вещественный
- любой вещественный -> любой целочисленный

#### Между alias и базовым типом

Разрешены тождественные приведения, хотя они и избыточны.

### 11.3. Запрещённые приведения

Запрещено приводить:

- `bool` к числовым типам и обратно;
- `string` к числовым типам и обратно;
- `string` к `bool` и обратно;
- одну структуру к другой;
- один массив к другому типу массива;
- `unit` к любому другому типу и обратно.

### 11.4. Семантика narrowing и widening

Обе категории преобразований допустимы **только явно**.

Примеры:

```txt
let a: int64 = 10 as int64;      // widening
let b: int8 = 300 as int8;       // narrowing, разрешено спецификацией
let c: int32 = 3.9 as int32;     // float -> int, разрешено явно
```

### 11.5. Переполнение при приведениях

Если при явном приведении значение не представимо в целевом целочисленном типе,
используется **арифметика по модулю 2^N** для целевого размера `N`.

То есть биты усекаются до младших `N` бит.

Это правило фиксирует поведение явно и снимает неоднозначность реализации.

Для приведения `float -> int` дробная часть отбрасывается в сторону нуля.
Если итоговое целое не помещается в целевой тип, далее применяется правило усечения по модулю.

---

## 12. Переполнение арифметики

### 12.1. Целочисленные типы

Для операций над целочисленными типами поведение при переполнении фиксируется так:

- для знаковых и беззнаковых целых используется **арифметика по модулю 2^N**,
  где `N` — размер типа в битах.

Примеры:

- `255 as uint8 + 1 as uint8 == 0 as uint8`
- `127 as int8 + 1 as int8 == -128 as int8`

### 12.2. Вещественные типы

Для `float32` и `float64` используется стандартная семантика IEEE 754 платформы исполнения.

---

## 13. Специальные типовые правила для строк

### 13.1. Допустимые операции над `string`

Для `string` определены:

1. конкатенация через `+`
2. сравнение `==` и `!=`
3. получение длины через встроенную функцию `len`

Для полноты типовой системы в языке вводится встроенная функция:

```txt
len(value: string) -> int32
```

Она является допустимым встроенным инструментом реализации строк,
не противоречащим базовому ТЗ.

### 13.2. Недопустимые операции над `string`

Запрещены:

- вычитание, умножение, деление, остаток;
- порядковые сравнения `< <= > >=`;
- неявные преобразования в числовые типы.

---

## 14. Типизация встроенных функций

Встроенные функции имеют следующие сигнатуры:

```txt
print(value: Printable) -> unit
input() -> string
exit(code: int32) -> unit
panic(message: string) -> unit
len(value: string) -> int32
```

`Printable` здесь — спецификационное обозначение множества допустимых типов для печати.
В базовой реализации это:

- все числовые типы;
- `bool`;
- `string`.

Это не отдельный пользовательский тип языка, а правило проверки аргумента встроенной функции.

---

## 15. Типизация инструкций

### 15.1. Условие `if`

Условие должно иметь тип `bool`.

### 15.2. Условие `while`

Условие должно иметь тип `bool`.

### 15.3. Присваивание

Если левая часть имеет тип `T`, то правая часть обязана иметь тип `T`.

### 15.4. Return

В функции с результатом `R`:

- `return expr;` требует, чтобы `expr` имело тип `R`;
- `return;` допустим только если `R = unit`.

---

## 16. Типовые ошибки

Компилятор обязан отклонять как минимум следующие программы:

1. использование операции с неподходящим типом;
2. присваивание значения несовместимого типа;
3. вызов функции с неверным числом аргументов;
4. вызов функции с неверными типами аргументов;
5. использование массива как скаляра и наоборот;
6. индексирование не-массива;
7. обращение к полю не-структуры;
8. обращение к несуществующему полю;
9. недопустимое приведение типов;
10. возврат значения неверного типа;
11. использование результата типа `unit` как обычного значения;
12. попытка вывести тип переменной там, где это не предусмотрено спецификацией.

---

## 17. Примеры корректных и некорректных программ

### 17.1. Корректно

```txt
struct Point {
    x: int32;
    y: int32;
}

fn main() -> int32 {
    let a = 10;
    let b: int32 = 20;
    var c: int32 = a + b;
    let p = Point { x: c, y: 0 };
    print(p.x);
    return 0;
}
```

### 17.2. Ошибка: нет неявного widening

```txt
fn main() -> int32 {
    var x: int64 = 10;
    return 0;
}
```

Причина: `10` имеет тип `int32`, а не `int64`.

### 17.3. Корректно с явным приведением

```txt
fn main() -> int32 {
    var x: int64 = 10 as int64;
    return 0;
}
```

### 17.4. Ошибка: разные структуры не совместимы

```txt
struct A { x: int32; }
struct B { x: int32; }

fn main() -> int32 {
    let a = A { x: 1 };
    let b: B = a;
    return 0;
}
```

### 17.5. Ошибка: запрещённый cast

```txt
fn main() -> int32 {
    let s = "123";
    let x = s as int32;
    return 0;
}
```

## 18. Итоговые свойства системы типов

Система типов языка **Astra** обладает следующими свойствами:

- статичность;
- строгая типизация;
- номинальная совместимость;
- отсутствие неявных приведений;
- детерминированные правила вывода типов для `let`;
- фиксированная семантика переполнения и явных преобразований;
- простота реализации в рамках семестрового компилятора.
